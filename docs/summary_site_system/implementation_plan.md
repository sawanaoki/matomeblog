# 目的
はちま起稿のようなまとめサイトを構築・運用するためのシステムを実装する。フロントエンドはAstroで構築し、バックエンド・情報収集ツールはNode.jsで実装する。

# 実装計画

## プロジェクト構成
- `frontend/` : Astroを用いた高速なフロントエンド
- `scraper/` : 情報収集用のNode.jsスクリプト・バッチ処理

## フェーズ 1: バックエンド・データ収集システムの実装 (scraper/)
- [ ] Node.js プロジェクトのセットアップ
- [ ] RSS取得機能の実装 (RSS-parser)
- [ ] X (Twitter) のトレンドまたは特定キーワード取得 (X API利用想定モックまたはスクレイピング)
- [ ] AI連携機能の実装 (OpenAI API等を用いたタイトル生成・要約モジュール)
- [ ] 収集データのローカル保存 (JSON) または データベース連携 (Supabase等の基本枠組み)

## フェーズ 2: フロントエンドの実装 (frontend/)
- [ ] Astro プロジェクトのセットアップ
- [ ] 収集したデータを表示する UI の作成 (まとめサイト風のデザイン)
- [ ] 記事詳細ページの静的生成 (SSG)

## フェーズ 3: ワークフロー・定期実行の枠組み
- [ ] 定期実行スクリプトの作成 (自動情報収集)
- [ ] 管理・公開フローのドキュメント作成

## ユーザー確認事項
- データベースは本格的なRDB（Supabaseなど）を使うか、まずは軽量なローカルJSONファイルベースで動かすか？
- AI API (OpenAIなど) のキーを持っていますか？
- フロントエンドはAstroでセットアップしてよろしいでしょうか？
